Temporal Coverage
The temporal coverage of the data set.
Start Date/Time The starting date/time of the temporal coverage of the data set, in ISO format.

Example: 2002-08-30T00:00:00Z
End Date/Time The ending date/time of the temporal coverage of the data set, in ISO format. If new data files are still being processed and/or archived, this value should be empty.

Example: 2011-06-29T22:29:59.000Z
Frequency The frequency of data files in terms of the temporal coverage interval. (Deprecated; set the value to 1.)
Interval The interval between the start and end times of a data set's files, e.g. "hourly" for data sets with a duration of 1 hour.

Example: hourly
Start time offset (seconds) The offset (in seconds) between the starting date/time of temporal coverage for each of a daily data set's data files and 00:00:00 UTC. This value should be 0 if the temporal coverage interval is anything other than "daily", or when the temporal coverage of a daily data file starts on a day boundary, and should be a value between -86,399 and 86,399 for daily files that start at other times.

Example: TRMM 3B42 Daily files start at 22:30:00 the day before the nominal "day of coverage", and thus have an offset of -5400 (seconds).
End time offset (seconds) The offset (in seconds) between the ending date/time of temporal coverage for each of a daily data set's data files and 00:00:00 UTC. This value should be 0 if the temporal coverage interval is anything other than "daily", or when the temporal coverage of a daily data file ends before the day boundary, and should be a value between -86,399 and 86,399 for daily files that end at other times.

Example: TRMM 3B42 Daily files end at 22:29:59 on the nominal "day of coverage", and thus have an offset of -5401 (seconds).
Duration of search intervals (in days) An optional value that determines the Giovanni "chunking" duration. Normally this value can be left empty.

Example: For the 'MISR Level 3 Component Global Aerosol Product covering a day' data set data variables, the value is 90.0 (days).

EDDA Data Variable Attributes
Title Description
Long Name The long name for the data variable. This is the name that will be displayed in the data picker, so systematic naming will help insure that similar data variables appear together in the data picker.

Example: For MOD08_D3 and MYD08_D3, example values are:
  • Aerosol Optical Depth 550 nm (Dark Target)
  • Angstrom Exponent 470/660 nm (Dark Target), Land-only
An example ordering of the long name constituents is:
  1. measurement
  2. wavelengths (or other "third dimension", if any) with units
  3. qualifiers
Variable Name The name of the variable(s) used for the chosen Access Method. If the Access Method is "OPeNDAP", then this is the name of the variable in OPeNDAP. If more than one variable is needed to represent the data variable, the names of the variables should be separated by commas [For example, the AIRX3STD Atmospheric Temperature profile needs both Temperature_A and TempPrsLvls_A, so the value in that case should be 'Temperature_A,TempPrsLvls_A']. If a variable has multiple dimensions, the name should also include the index of the extra dimensions. Choose one value.

Example: For MOD08_D3, the Access Method is "OPeNDAP", and the name of the ocean-only Angstrom Exponent variable in OPeNDAP is Angstrom_Exponent_1_Ocean_QA_Mean.
For MIL3DAE 004, there are also OpticalDepth, Band, and ParticleType dimensions, so to choose the first optical depth, the second band, and the first ParticleType, the name should be Optical_depth_average/OpticalDepth[0]/Band[1]/ParticleType[0]
Color Palettes The color palettes used to display plots of this data variable. (Often a palette related to the data variable's measurement already exists; if not, a new palette may need to be created.) Choose one or more.

Example: For the MOD08_D3 data variable Angstrom_Exponent_1_Ocean_QA_Mean, the "Angstrom Exponent" palette is the best choice.
Measurements The main quantity being measured by the data variables. Choose one or more values.

Example: Aerosol Index
Description URL The URL for a web page or a portion of a web page that describes the data variable. The data picker will use this URL to link the data variable to its description. Choose one value.

Example: Angstrom Exponent data variables might use the URL https://disc.gsfc.nasa.gov/information/glossary?keywords=giovanni%20measurements&title=Giovanni%20Measurement%20Definitions:%20Angstrom%20Exponent
Disciplines The science discipline associated with this data variable. Choose one or more values.

Example: Aerosols
Keywords Keywords and keyword phrases that can be used to find the data variable via a keyword search in Giovanni. One of the keywords should always be the shortname of the data set. Use the "Add Keywords" button to add additional keywords, and use the "Remove" button to remove keywords.

Words and phrases from the values of the following fields will automatically be used in keyword searches even if they are not added as keywords :
  • (Data Set) Short Name
  • (Data Set) Platform Short Name
  • (Data Set) Platform Long Name
  • (Data Set) Instrument Short Name
  • (Data Set) Instrument Long Name
  • Variable Name
  • Measurements
  • CF-1 Standard Name
  • Special Features
Tags One or more tags (text strings) that can be used to produce bookmarkable URLs that will only display data variables with that tag in Giovanni. These tags can be used to create virtual groups of related data variables, i.e. portals. Use the "Add Tags" button to add additional tags.

Example: Basic
Granule Search Results Filter (regex) A filter (in the form of a regular expression) that is applied to the results of an OpenSearch for granules. It is used to exclude granules from the results returned to Giovanni. Only granules with a download link which matches the regular expression will be included.

No value is needed for this attribute when the OpenSearch result will only contain links for the data set whose attributes (such as spatial resolution and temporal coverage) were specified in the EDDA Data Set Attributes. However, there may be some cases when the OpenSearch result contains granules whose attributes do not match what was specified for the data set (e.g. there is more than one spatial resolution), but they can be distinguished via pattern matching, and in those cases a value should be specified for this attribute. The value should be specified as a regular expression.

Example: opendap.+L3m_MO_SST4_sst4_4km can be used to select only granules with a download link that contains 'opendap' followed by one or more characters before the string 'L3m_MO_SST4_sst4_4km'.
CF-1 Standard Name The CF-1 Metadata compliant "Standard Name" for the data variable. See CF Standard Names. Since not every data variable has a corresponding standard name, this is often left empty.

Example: air_temperature
Name of fill value attribute The name of the fill value variable in the data file.

Example: _FillValue
Units The units associated with this data variable. If the data variable is unitless, this is often the value '1'. Choose zero or one value.

Example: m/s
Allowed Units Conversions The conversions that are allowed for the units associated with this data variable. Choose zero or more values.

Example: mm/s and mm/hr
Deflation Level The deflation level, needed only if the data files are compressed NetCDF. Otherwise it should be left empty.

Example:1
Virtual data variable generator Usually not used. In some cases, a "virtual" data variable can be generated by a formula for the ncap2 equation that produces a single value from one or more variables. The left side of the equation should be the data variable id.

Example: The value of air pressure at sea level from the MAT1NXSLV data set can be rescaled via the formula MAT1NXSLV_5_2_0_SLP=SLP/100.
Vector Component Names An optional field that only needs a value if the variable represents a vector, in which case it should be the comma-separated names of all of the vector component variables. Choose one value.

Example: For the NLDAS_FORA0125_H.002 data set, the wind velocity vector has two components, which are zonal wind speed and meridional wind speed, so the value used is the comma-separated list of the names of those two variables, which is "N10-m_above_ground_Zonal_wind_speed,N10-m_above_ground_Meridional_wind_speed"
Can be accumulated? Set to "true" to indicate that values of this data variable can be accumulated over time, e.g. accumulated precipitation amounts. Only set this to "true" for data variables for which it makes sense to accumulate the measurement.
Minimum Valid Value The minimum valid data value. This is an optional value that should be set when the data contains invalid values; setting the minimum value will cause values less than the minimum to be ignored.

Example: The MOD08_D3 0051 Deep Blue AOD at 550 nm from LAADS has values less than 0, so setting this value to '0.' will cause those values to be ignored.
Maximum Valid Value The maximum valid data value. This is an optional value that should be set when the data contains invalid values; setting the maximum value will cause values greater than the maximum to be ignored.

Example: The MOD08_D3 051 Deep Blue AOD at 550 nm from LAADS has values greater than 5, so setting this value to '5.' will cause those values to be ignored.
Nominal minimum value The nominal minimum data value. This is an optional value that specifies the smallest nominal value expected in the data. All values smaller than this will be rendered in a map plot as the same color.

Example:
Nominal maximum value The nominal maximum data value. This is an optional value that specifies the largest nominal value expected in the data. All values larger than this will be rendered in a map plot as the same color.

Example:
Data Values Distribution The distribution of the data values, such as "linear" or "log(arithmic)". The value chosen will determine the distribution of palette colors and the scale displayed for the palette in map plots.

Example: linear
Time Interval Representative Position The position in the interval between the start and end times of a data set's file that represents the "data time" for a variable, which is either the start, middle, or end of the interval.

Example: start
Wavelengths If there are wavelengths associated with the data variable, a (numerical) wavelength value should be provided for each wavelength. No values need to be provided if there are no wavelengths associated with the data variable. (Units will be specified in "Wavelength Units"). Provide zero or more values.

Example: For MOD08_D3, the ocean-only Angstrom Exponent variable has two wavelength values associated with it, "550" and "865".
Wavelength Units If there are wavelengths associated with the data variable, the units for the wavelength values. Provide zero or more values.

Example: For MOD08_D3, the ocean-only Angstrom Exponent variable has "nm" as the wavelength units.
Depths The range of depths below the surface for the data measurement. Provide zero or one value.

Example: 0-10
Depth Units Units for the range of depths below the surface for the data measurement. Provide zero or one value.

Example: cm
Z-Dimension name (if any) If this data variable has a Z-dimension, the name used for the Z-dimension values. Choose one value if the data variable has a Z-dimension, otherwise leave it empty.

Example: TempPrsLvls_A
Z-Dimension type (if any) If this data variable has a Z-dimension, the type ('pressure', 'altitude', 'depth') of the Z-dimension values. Choose one value if the data variable has a Z-dimension, otherwise leave it empty.

Example: pressure
Z-Dimension units (if any) If this data variable has a Z-dimension, the units used for the Z-dimension values. Choose one value if the data variable has a Z-dimension, otherwise leave it empty.

Example: hPa
Z-Dimension values (space-separated, if any) If this data variable has a Z-dimension, the order, space-separated list of Z-dimension values. Provide a list of values if the data variable has a Z-dimension, otherwise leave it empty.

Example: For the case where the Z-dimension is "pressure", the list of values (in hPa) might be "1000 500 100 50 30 10 5 1".
